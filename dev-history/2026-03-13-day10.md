# Day 10

## Summary

Connected the Whiteboard org builder to real Hats Protocol state on Sepolia, persisted that org metadata in Neon, and finished the smart-wallet sponsorship path so studio and role creation can run as actual onchain writes instead of pure local demo state.

## Key Changes

- Naming a studio now creates or updates a player-owned Hats tree, with the player execution wallet wearing the top hat.
- Creating Whiteboard roles now creates real child hats under the studio tree, while worker assignment remains offchain for now.
- Added server-backed persistence for org trees and role hats so returning players restore the same org structure during bootstrap.
- Wired the Privy smart-wallet execution path behind configuration and validated the sponsored Sepolia flow with Pimlico-backed smart-wallet settings.
- Kept replay behavior aligned with the prototype: resetting the demo preserves the existing Hats tree and role hats, and a renamed studio updates the same top-hat metadata instead of minting a new tree.
- Added a small amount of non-cluttered org context to Whiteboard by surfacing the chain label and decimal Hats tree ID on the studio root card.
