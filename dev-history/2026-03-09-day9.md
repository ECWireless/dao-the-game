# Day 9

## Summary

Added the first real player identity layer to the demo with Privy auth, server-backed player bootstrap, persisted progress tracking, and player-scoped run restoration while keeping the tutorial flow intact.

## Key Changes

- Added Privy login with both email and external wallet entry points, while still creating an embedded wallet for every user so gameplay can keep a consistent execution wallet.
- Introduced Vercel API routes and Neon persistence for `players`, `progress_events`, `progress_summary`, and `game_state`.
- Added TanStack Query for authenticated bootstrap, progress writes, game-state sync, and reset mutations.
- Reworked onboarding so sign-in happens inside the intro brief and authenticated players drop straight into the first beat.
- Replaced the old floating session controls with a header menu for `Reset demo` and `Sign out`, and tuned the menu styling to match the app chrome.
- Updated replay/reset behavior so `Play Again` returns the player to Messages and preserves analytics state correctly.
- Added a `pnpm reset:all` script to wipe Neon app data and make full local resets easier during development.
- Prepared Day 2 smart-account work by enabling Privy smart accounts against Base Sepolia without yet wiring paymaster or bundler settings in-app.
