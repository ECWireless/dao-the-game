# Day 11

## Summary

Refactored the Machine from a single blended scoring function into an explicit execution pipeline, while keeping the tutorial’s forced fail/success beats intact and preserving the current demo flow.

## Key Changes

- Added a real pipeline model with ordered stages: `design`, `implementation`, `review`, and `deployment`.
- Separated org structure from execution order by introducing explicit stage mappings for tutorial roles instead of relying on tree layout to imply workflow.
- Reworked run simulation so worker assignments now influence specific pipeline stages, with missing coverage producing blocked stages rather than only lowering one global score.
- Kept the top-level run surface compatible, so artifacts, previews, and tutorial shaping still work without rewriting the surrounding game flow.
- Updated the Machine scene to read stage-aware execution data for its status copy while keeping the visual presentation largely unchanged.
- Added test coverage for stage ordering, under-configured pipelines, and compatibility with the tutorial’s forced first-run failure and second-run success.
- Cleaned up the top-right player menu by removing the duplicated embedded wallet address and loosening the mobile spacing so the menu feels less cramped on iPhone.
