# Day 18

## Summary

Built the first real Day 4 worker registry pass for Sprint 3. DAO the Game now publishes a canonical worker network config, tells workers to use Base Sepolia for the current registration and payment setup phase, accepts public worker submissions into a Neon-backed registry, and exposes a small read-only registry surface for inspection.

## Key Changes

- Added DAO-owned worker network config under `public/.well-known/dao-the-game/networks.json`, with Base Sepolia set as the active registration and payment network for the current testing phase.
- Added `public/.well-known/dao-the-game/schemas/networks.v1.json` so agents can validate the network config shape.
- Extended `public/workers.md` with:
  - network rules sourced from `networks.json`
  - a Pinata `@pinata/erc-8004` registration step after preview approval
  - Base Sepolia guidance for both ERC-8004 and payment setup
  - a public DAO registry submission step after onchain registration
- Added public registry submission schemas:
  - `registry-submit-request.v1.json`
  - `registry-submit-response.v1.json`
- Added a new worker registry contract in `src/contracts/workers.ts`.
- Added a new `worker_registry` table to the DB migration script.
- Added Neon-backed worker registry helpers in `api/_lib/db.ts`.
- Added shared Day 4 worker validation helpers in `api/_lib/workerRegistry.ts`:
  - worker origin normalization
  - live manifest/profile/self-test fetch + validation
  - Base Sepolia ERC-8004 owner and token URI verification
- Added public worker registry API routes:
  - `GET /api/workers`
  - `POST /api/workers`
  - `GET /api/workers/:id`
- Added `public/workers-admin.html` as a minimal read-only registry/debug view.

## Notes

- The registry is intentionally public for now.
- `/run` is still allowed to remain public during the current testing phase.
- Day 6 should flip worker guidance from Base Sepolia to Base mainnet.
