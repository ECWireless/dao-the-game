# Day 15

## Summary

Started Sprint 3 by defining a real worker standard instead of treating the built-in roster as bespoke app data. The demo now has a canonical worker manifest, a local dummy registry shaped like the future global registry, role-tag metadata on hats, and a repo-wide shift from `Agent` language to `Worker` language without breaking the current Assembler flow.

## Key Changes

- Added a canonical `WorkerManifest` and `WorkerRegistryRecord` shape, plus supporting `EngineerIntake`, `RoleTagId`, and `HatMetadata` types.
- Introduced a central role-tag registry for the v1 taxonomy:
  - `ui-designer`
  - `brand-designer`
  - `frontend-engineer`
  - `code-reviewer`
- Moved the six built-in hires into a local dummy worker registry that now carries:
  - manifest data
  - synthetic registration data
  - availability state
  - local presentation metadata
  - local gameplay metadata for the current sim
- Renamed the repo’s core `Agent` model to `Worker` across state, sim, artifact generation, and UI plumbing.
- Added hat metadata for role matching and future tree execution:
  - `allowedRoleTagIds`
  - `pipelinePath`
  - `executionMode`
- Updated RaidGuild candidate filtering so hats now match workers by allowed role tag first, while keeping current stage scoring only as a local demo tiebreaker.
- Bumped persisted gameplay storage to a new key/version to match the renamed worker-based state model.
- Replaced the old internal contract-cost spend model with pricing derived from each worker manifest’s USDC per-request license amount.
- Simplified the worker profile card so it no longer teaches the player internal score grids and strategy labels that don’t belong in the canonical worker standard.
